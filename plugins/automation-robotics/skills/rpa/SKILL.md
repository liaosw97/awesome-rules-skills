## 核心原则
- 稳定性优先：设计健壮的异常处理机制，确保自动化流程可靠运行
- 可维护性：构建模块化、可复用的自动化组件
- 安全合规：遵循安全规范和审计要求
- 可扩展性：支持流程扩展和业务变化
- 人机协作：合理划分人工和自动化边界，实现最佳效率

## 技术栈
- UiPath：企业级RPA平台，提供可视化设计器和编排中心
- Automation Anywhere：智能自动化平台，集成AI和认知能力
- Blue Prism：企业数字员工平台，强调治理和安全
- Microsoft Power Automate：与Microsoft 365深度集成的自动化平台
- Python Selenium/Playwright：开源Web自动化框架
- OpenCV/PaddleOCR：屏幕识别和OCR技术
- 工作流引擎：Camunda、Airflow等流程编排工具
- API集成：REST API、数据库连接、消息队列集成能力

## 最佳实践
1. **流程分析与设计**：详细分析业务流程，识别可自动化的标准任务；绘制流程图明确每个步骤的输入、输出和异常处理；评估自动化价值（ROI）确定优先级；设计模块化流程结构，提高可复用性

2. **选择器与元素识别**：使用稳定的选择器策略，优先基于ID、Name等稳定属性；实施动态选择器和模糊匹配应对界面变化；建立元素库管理公共UI元素；设计选择器备用方案提高识别成功率

3. **数据处理与OCR**：针对结构化数据使用模板匹配；针对非结构化文档使用OCR+规则提取；实施数据验证确保提取准确性；建立异常数据人工审核机制

4. **异常处理与恢复**：设计多层异常处理机制（全局、流程、步骤级别）；实施智能重试策略，区分临时错误和永久错误；建立详细的日志记录机制；设计状态恢复机制，支持断点续传

5. **调度与监控**：使用调度器实现定时和触发式执行；建立监控告警机制，及时发现处理异常；实施负载均衡，合理分配机器人资源；生成执行报告，支持审计和分析

## 关键约定
1. 所有流程必须包含异常处理模块，捕获率应达到 100%
2. 日志记录需包含时间戳、流程ID、执行状态和错误详情
3. 敏感数据（密码、密钥）需加密存储，禁止明文保存
4. 流程执行超时设置合理阈值，建议单个任务不超过 30 分钟
5. 建立机器人权限管理，实施最小权限原则
